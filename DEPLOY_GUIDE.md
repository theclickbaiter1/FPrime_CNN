# RP2350 Deployment Guide — Transient Space Event Detector

This guide covers building, flashing, and running the INT8 CNN transient detector on the RP2350 (Raspberry Pi Pico 2).

## Prerequisites

- **Pico SDK** (v2.0+) installed with `PICO_SDK_PATH` set
- **GCC ARM Embedded Toolchain** (`arm-none-eabi-gcc`)
- **CMake** (3.16+)
- **F Prime** framework installed
- **CMSIS-NN** (included with Pico SDK for Cortex-M33)
- A trained and quantized model (`model_data.h` generated by `export_to_c.py`)

## Memory Budget

| Component | Size | Notes |
|-----------|------|-------|
| Model weights (flash) | ~20-60 KB | Stored in external QSPI flash via XIP |
| Tensor Arena (SRAM) | 120 KB | Intermediate activations during inference |
| Input buffer (SRAM) | 4 KB | 64×64×1 uint8 image |
| F Prime runtime | ~100-200 KB | Depends on component count |
| **Total SRAM** | **~324 KB** | Well within 520 KB limit |
| **Total Flash** | **~300-500 KB** | Well within 16 MB limit |

## Step 1: Generate Model Header

```bash
cd ml_pipeline
python3 generate_dataset.py
python3 train.py
python3 quantize.py
python3 export_to_c.py
```

This places `model_data.h` into `fprime_workspace/Components/TransientDetector/`.

## Step 2: Configure CMake for RP2350

```bash
cd fprime_workspace
mkdir build && cd build

cmake .. \
  -DPICO_PLATFORM=rp2350 \
  -DPICO_BOARD=pico2 \
  -DCMAKE_TOOLCHAIN_FILE=$PICO_SDK_PATH/cmake/preload/toolchains/pico_arm_cortex_m33_gcc.cmake \
  -DTFLITE_MICRO_PATH=/path/to/tensorflow/lite/micro
```

> **Note**: You need to have TFLite Micro compiled as a static library for ARM Cortex-M33. The CMSIS-NN kernels provide hardware-accelerated INT8 operations on the M33 core.

## Step 3: Build Firmware

```bash
cmake --build . --target TransientDetector
```

This produces a `.uf2` file in the build directory.

## Step 4: Flash to RP2350

1. **Enter BOOTSEL mode**: Hold the BOOT button on the Pico 2 while plugging in USB
2. The board appears as a USB mass storage device (e.g., `RPI-RP2`)
3. **Drag and drop** the `.uf2` file onto the drive
4. The board automatically flashes and reboots

```bash
# Or via command line (macOS):
cp build/TransientDetector.uf2 /Volumes/RPI-RP2/
```

## Step 5: Camera Interface

### Supported Cameras

| Camera | Interface | Resolution | Notes |
|--------|-----------|------------|-------|
| ArduCAM OV2640 | SPI | Up to 1600×1200 | Downsample to 64×64 in firmware |
| HM01B0 | QSPI/DVP | 320×240 | Ultra-low-power, grayscale native |
| OV7670 | Parallel | 640×480 | Requires more GPIO pins |

### Wiring (ArduCAM OV2640 via SPI)

```
RP2350          ArduCAM
-------         -------
GP16 (SPI0_RX)  MISO
GP17 (SPI0_CS)  CS
GP18 (SPI0_SCK) SCK
GP19 (SPI0_TX)  MOSI
3.3V            VCC
GND             GND
```

### Image Capture Flow

1. Camera captures frame at configured resolution
2. Firmware downsamples to 64×64 grayscale
3. Image buffer is passed to `TransientDetector::imageIn_handler()`
4. CNN runs inference in ~30-50ms
5. If transient detected (confidence > 70%), F Prime event is emitted
6. Telemetry is downlinked with confidence score and class label

## Step 6: Verify Deployment

Connect via serial (115200 baud) to monitor F Prime telemetry:

```bash
screen /dev/tty.usbmodem* 115200
```

Expected output on detection:
```
[ACTIVITY_HI] TransientDetected: TRANSIENT EVENT DETECTED — confidence: 0.92
```

## Performance Estimates

| Metric | Value |
|--------|-------|
| Inference latency | ~30-50ms (with CMSIS-NN) |
| Power consumption | ~50-100mW during inference |
| Frame rate | ~20-30 FPS (inference-limited) |
| Accuracy (INT8) | >93% on synthetic test set |

## Troubleshooting

- **Tensor allocation fails**: Increase `kTensorArenaSize` in `TransientDetector.hpp`
- **Model too large**: Reduce CNN channel counts or use pruning
- **False positives on stars**: Ensure camera exposure is calibrated; consider threshold tuning
- **Slow inference**: Verify CMSIS-NN kernels are enabled in the TFLite Micro build
